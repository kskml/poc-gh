Post-Processing Overlap Detection: Write a simple script to detect bounding box overlaps in the rendered diagram (using a headless browser or image analysis). If detected, flag it for a second LLM pass focused only on layout adjustment.

Implementation Steps:
Generator Agent: Your current LLM call that produces the initial gap analysis and diagram.
Validation Layer: This is the core addition. Run three checks in parallel or sequence:
Syntax Validation: Use a Python library to validate the Mermaid code. On failure, pass the diagram and error to a Syntax Repair LLM with a focused prompt ("Fix the syntax errors in this Mermaid code: [error message]").
Accuracy Critique: Use a second LLM (can be a smaller, cheaper model) as the Critic. Provide it with the original section, code summary, and the Generator's output. Prompt: "Critique this analysis for accuracy and completeness. List any missed gaps or incorrect actions." Feed this critique back to a Refinement LLM to improve the analysis.
Layout Analysis: Use a simple script to parse the diagram nodes and edges. Run a basic overlap detection algorithm. If issues are found, pass the diagram to a Layout Adjuster LLM with a prompt focused solely on improving spacing and clarity.
Orchestrator: A simple script that manages this pipeline, handling retries and aggregating the final, validated output.
